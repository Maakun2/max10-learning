# FPGA LED Moving Project

## 概要

MAX10 FPGA を用いて，LED が左右に移動する回路を作成した。

今回作成した動作は以下のような往復型である。

```text
001
010
100
010
001
...
```

この課題では，単純なLED制御だけでなく，

- シフト演算
- 状態保持
- 順序回路
- フラグによる状態管理

など，FPGA/Verilog の基礎的な考え方を学習した。

---

# 使用した環境

- FPGA : Intel MAX10
- 開発環境 : Quartus Prime Lite Edition
- 言語 : Verilog HDL
- クロック : 48MHz

---

# 実装した回路

## 1秒周期生成回路

48MHz クロックをカウントして1秒周期信号を生成した。

```verilog
`define CYCLE_1SEC 48000000
```

```verilog
assign period_1sec = (counter_1sec == (`CYCLE_1SEC - 1));
```

ここで `-1` をしている理由は，

```text
0 から数え始めるため
```

である。

例えば 48000000 回数える場合，

```text
0 ～ 47999999
```

となるため，最後は `48000000 - 1` になる。

# LED移動回路

## 最初に考えたこと

最初は，

- 足し算
- 掛け算

などでLEDのパターンを作ろうとしていた。

しかし，

```text
001
010
100
```

というビットの移動は，Verilogではシフト演算を使う方が自然であると気づいた。

---

# シフト演算

## 左シフト

```verilog
counter_led <= counter_led << 1;
```

これは以下のように動作する。

```text
001 -> 010
010 -> 100
```

ただし，

```text
100 << 1 = 1000
```

となり，3bitを超えてしまう。

そのため，

```verilog
if (counter_led == 3'b100)
    counter_led <= 3'b001;
```

として先頭へ戻す必要があった。

---

# ループ型と往復型

最初は，

```text
001 -> 010 -> 100 -> 001
```

というループ型を考えていた。

しかし課題の意図を考え直した結果，

```text
001 -> 010 -> 100 -> 010 -> 001
```

という往復型であることに気づいた。

この違いを考えることで，「状態」の重要性を理解できた。

---

# 状態を持つという考え方

往復型では，

```text
010
```

という状態が2回登場する。

しかし，

```text
最初の010 -> 次は100
帰りの010 -> 次は001
```

となる。

つまり，

```text
現在のLED状態だけでは次の状態を決められない
```

ということが分かった。

そのため，

```text
今どちら向きに進んでいるか
```

を保持する必要がある。

そこで，

```verilog
reg direction;
```

というフラグを追加した。

---

# direction フラグ

```verilog
direction = 0
```

のとき：

```text
左から右へ進む
001 -> 010 -> 100
```

```verilog
direction = 1
```

のとき：

```text
右から左へ戻る
100 -> 010 -> 001
```

となる。

端まで到達したときに direction を反転させることで往復動作を実現した。

# 学んだこと

## 1. シフト演算の重要性

最初は算術演算にとらわれていたが，

```verilog
<<
>>
```

を使うことでビット移動を自然に表現できることを学んだ。

---

## 2. FPGAでは状態が重要

今回の課題では，

```text
LEDの値
```

だけでなく，

```text
現在どちら向きに進んでいるか
```

という状態を保持する必要があった。

これにより，

```text
順序回路
```

の考え方を体験できた。

---

## 3. assign と always の違い

### assign

```verilog
assign led = ~counter_led;
```

は，

```text
常時接続された配線
```

のような動作をする。

---

### always

```verilog
always @(posedge clk)
```

は，

```text
クロックに同期して状態を更新する
```

ための記述であり，状態保持を行う。
