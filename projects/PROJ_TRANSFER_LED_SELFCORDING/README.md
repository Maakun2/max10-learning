# FPGA学習ログ：往復型LED制御と補間処理モジュール理解

## 今日の目的

本日は，

* Verilog HDL による往復型LED制御
* shift演算
* direction flag
* パイプライン処理
* 補間処理モジュール

について学習し，

最終的に研究テーマである

「スペクトル拡散超音波受信機回路」

との接続を理解することを目的とした．

---

# 1. 往復型LED制御

## 実装した動作

```text
001 → 010 → 100 → 010 → 001
```

LEDを左右へ往復させる回路を作成した．

---

# 2. direction flag の役割

往復動作では，

```text
現在どちら方向へ進んでいるか
```

を保持する必要がある．

そのため，

```verilog
reg direction;
```

を用いて，

* `0` ：左方向
* `1` ：右方向

を表現した．

---

# 3. shift演算の理解

## 左シフト

```verilog
counter_led <= counter_led << 1;
```

これは，

```text
bitを左へ1つ移動
```

させている．

### 動作例

```text
001 → 010 → 100
```

---

## 右シフト

```verilog
counter_led <= counter_led >> 1;
```

### 動作例

```text
100 → 010 → 001
```

---

# 4. FPGAは並列回路

CPUでは，

```c
A();
B();
C();
```

のように逐次実行される．

しかしFPGAでは，

```verilog
always @(posedge clk)
```

ごとに独立した回路が生成される．

つまり，

```text
always文 = 回路
```

である．

---

# 5. counter と period の役割

## counter

```verilog
counter_1sec
```

はクロック数を数える．

---

## period

```verilog
period_1sec
```

は，

```text
1秒経過したか
```

を判定する信号である．

```verilog
assign period_1sec =
(counter_1sec == (`CYCLE_1SEC - 1));
```

によって生成される．

---

# 6. 非同期リセット

```verilog
always @(posedge clk, negedge res_n)
```

を使用した．

## 意味

* clk立ち上がりで動作
* res_n立ち下がりで即リセット

---

# 7. Verilogで学んだ文法

## module宣言

```verilog
module FPGA(
    input wire clk,
    input wire res_n,
    output wire [2:0] led
);
```

---

## assign

```verilog
assign led = ~counter_led;
```

* assign は `=`
* `<=` ではない

---

## define

```verilog
`define CYCLE_1SEC 48000000
```

使用時：

```verilog
`CYCLE_1SEC
```

---

# 8. サンプルデータとは

超音波は本来連続波形である．

しかしFPGAでは，

```text
一定時間ごとの振幅値
```

として扱う．

例：

```text
12 18 25 20 10
```

これを：

```text
サンプルデータ
```

という．

---

# 9. 3次補間処理

## 補間とは

```text
途中の値を推測すること
```

である．

---

## 3次補間

4点のサンプルを用いて，

```text
滑らかな3次関数
```

を作成し，

```text
途中の値
```

を推定する．

---

# 10. リサンプリング

## 意味

```text
少しズレた位置の値
```

を新しく生成すること．

例：

```text
2番目
```

ではなく，

```text
2.1番目
```

の値を求める．

---

# 11. ドップラー補正

移動体ではドップラーシフトにより，

```text
波形が伸びたり縮んだりする
```

ため，

```text
元の40kHzへ戻す
```

必要がある．

そのために：

* 3次補間
* リサンプリング

を行う．

---

# 12. Δt の意味

Δt は，

```text
1サンプルあたりの時間ズレ
```

を表す．

---

## T + 2Δt の意味

補間は2サンプルごとに行うため，

```text
次の補間位置
```

は：

```text
T + 2Δt
```

となる．

---

# 13. シフトレジスタ

補間処理モジュールでは，

```text
7つのシフトレジスタ
```

を使用していた．

例：

```text
reg1 reg2 reg3 reg4 reg5 reg6 reg7
```

ここから4点を選択する．

---

# 14. なぜreg選択を変えるのか

補間点Tは，

```text
4点の中央付近
```

にあるほど精度が高い．

そのため：

## 通常

```text
reg2 reg3 reg4 reg5
```

---

## 右へ行きすぎた場合

```text
reg3 reg4 reg5 reg6
```

---

## 左へ行きすぎた場合

```text
reg1 reg2 reg3 reg4
```

へ変更する．

---

# 15. 3サンプル/1サンプル読み込み

## 通常

2サンプル読み込み．

---

## T+2Δt > 1

右へ進みすぎるため，

```text
3サンプル読み込み
```

して補正する．

---

## T+2Δt < -1

左へ戻した分を打ち消すため，

```text
1サンプル読み込み
```

する．

---

# 16. パイプライン化

補間処理モジュールは，

```text
流れ作業
```

のように動作する．

つまり，

* 今のデータを計算
* 次のデータを準備

を同時並行で行っている．

---

# 17. 今日最も重要だった理解

本日最も重要だった理解は，

```verilog
counter_led <= counter_led << 1;
```

が，

```text
単なるLED制御
```

ではなく，

```text
データ移動
```

そのものであること．

---

# 18. 研究との接続

LED制御：

```text
001 → 010 → 100
```

研究：

```text
12 18 25 20
↓
18 25 20 10
```

どちらも本質的には，

```text
データを横方向へシフト
```

している．

つまり，

```text
LED shift
↓
シフトレジスタ
↓
補間処理
↓
ドップラー補正
```

という接続を理解できた．
